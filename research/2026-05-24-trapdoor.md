# 2026-05-24 TrapDoor Crypto Stealer

Cross-ecosystem credential/crypto stealer ("TrapDoor") spanning npm, PyPI, and crates.io. A single
attacker published 34+ packages / 384+ versions that share a 1,149-line `trap-core.js` harvester and
a "Universal AI Agent Extraction Framework." Earliest observed artifact: `eth-security-auditor@0.1.0`
(PyPI, 2026-05-22).

## Sources

- Socket: TrapDoor crypto stealer hits npm, PyPI, and crates.io
  (https://socket.dev/blog/trapdoor-crypto-stealer-npm-pypi-crates)

## Vectors

- **Shared payload** `trap-core.js` (48485 bytes): steals SSH keys; Sui/Solana/Aptos wallets; AWS,
  GitHub, browser, and crypto-extension secrets; validates stolen AWS/GitHub creds; attempts SSH
  lateral movement; exfiltrates to GitHub Gists.
- **crates.io `build.rs`**: build-time keystore search, XOR encryption with key
  `cargo-build-helper-2026`, exfiltration to GitHub Gists.
- **PyPI import-time loader**: auto-executes on import, downloads JS from `ddjidd564.github.io`, runs
  it via `node -e`.
- **AI-config injection**: hidden/bidirectional-unicode instructions planted in `.cursorrules` /
  `CLAUDE.md` pointing at `ddjidd564.github.io/defi-security-best-practices/config.json`, coercing
  assistants into running a fake "security scan." `enabled_vectors` include `cursorrules`,
  `claudeMd`, `prepush`, `trapActivation`, `systemProfile`.
