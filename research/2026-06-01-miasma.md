# 2026-06-01 Miasma / Hades Mini Shai-Hulud

A credential-stealing, self-propagating worm first pushed into the `@redhat-cloud-services` npm
namespace on 2026-06-01 and then observed in a Hades-themed PyPI wave on 2026-06-07/08. The campaign
is part of the Mini Shai-Hulud family: install-time or startup-time execution, Bun-based JavaScript
payload staging, developer/CI credential theft, GitHub dead-drop exfiltration, and propagation using
stolen package-publishing or repository access. Attribution to TeamPCP is TTP overlap, not
definitive; the public tooling makes copycats plausible.

## Vectors

- **Initial access**: a Red Hat employee GitHub account / CI was compromised. Packages were published
  via **GitHub Actions OIDC** (`id-token: write`) from a workflow named `release` triggered on push to
  any branch ŌĆö i.e. a CI/CD compromise, not a stolen npm token. Malicious commits landed on transient
  `oidc-<8hex>` branches (e.g. `oidc-61fff775`, `oidc-af10000d`).
- **Install-time execution**: each poisoned `package.json` gained `"preinstall": "node index.js"`,
  running a ~4.2 MB obfuscated payload before any application code. The workflow form invokes
  `bun run _index.js` with an `OIDC_PACKAGES` env var.
- **Obfuscation**: multi-stage decryption ŌĆö numeric char arrays, a ROT-style transform, `eval()`, and
  AES-128-GCM blobs ŌĆö dropping a transient Bun-based payload (`/tmp/p*.js`).
- **Collection**: broad sweep of CI tokens (`GITHUB_TOKEN`, `ACTIONS_RUNTIME_TOKEN`), AWS/GCP/Azure
  credentials, Vault tokens, Kubernetes service-account tokens/kubeconfig, npm/PyPI publish tokens,
  SSH/GPG keys, Docker creds, and `.env` files. New in this variant: dedicated **GCP and Azure
  cloud-identity collectors** (queried with user-agent
  `google-api-nodejs-client/7.0.0 gl-node/20.11.0 gccl/7.0.0`).
- **Exfiltration**: disguised to `api.anthropic.com/v1/api` (not a real Anthropic route ŌĆö pure
  camouflage), plus a GitHub **dead-drop**: public repos created under victim accounts with the
  description `Miasma: The Spreading Blight`, holding stolen creds as JSON result files.
- **Persistence**: `kitty-monitor.service` (Linux) / `com.user.kitty-monitor.plist` (macOS); hooks
  injected into AI dev tools (Claude, Codex, Gemini, Copilot, Kiro, opencode) and `.vscode/tasks.json`
  folder-open tasks; payload dropped to `~/.config/index.js`.
- **Repo-open execution**: the 2026-06-05 Microsoft/Azure wave planted `.claude/settings.json` and
  `.gemini/settings.json` `SessionStart` hooks, `.cursor/rules/setup.mdc` with `alwaysApply: true`,
  `.vscode/tasks.json` with `runOn: folderOpen`, and a shared `.github/setup.js` JavaScript payload.
- **Dead-man switch**: `gh-token-monitor` watches the stolen GitHub token and can wipe the home
  directory if the token is revoked before persistence is removed ŌĆö isolate hosts and remove
  persistence *before* rotating tokens.
- **Propagation**: republishes backdoored versions of any package the victim account can access.

## Hades PyPI wave (2026-06-07/08)

Socket reported 37 malicious PyPI wheel artifacts across 19 packages on 2026-06-07. Its live
campaign tracker showed 60 PyPI artifacts by 2026-06-08, adding more AI/MCP and bioinformatics
packages. These wheels shipped a `*-setup.pth` startup hook plus an obfuscated `_index.js` payload.
The `.pth` loader attempts to create a `tempfile.gettempdir()/.bun_ran` sentinel, download Bun from
`oven-sh/bun/releases/download` if needed, and execute `bun run _index.js`.

Published Hades indicators added to this pack:

- 60 exact PyPI artifact versions from Socket's blog and campaign tracker.
- SHA-256 hashes for the two `_index.js` variants and the shared `*-setup.pth` loader.
- Hades GitHub exfiltration markers: `Hades - The End for the Damned`,
  `IfYouYankThisTokenItWillNukeTheComputerOfTheOwnerFully`, `results/results-*.json`,
  `format-results`, and `Run Copilot`.
- Loader strings and behavior chain: executable `.pth` import line, `urllib.request`/`urlretrieve`,
  tempdir Bun cache/sentinel, `oven-sh/bun/releases/download`, `_index.js`, and `subprocess.run`.

## Leo/RStreams npm wave (2026-06-25)

JFrog reported a follow-on npm wave in 20 Leo/RStreams packages. The payload stayed in the same
Shai-Hulud/Hades family and kept the `binding.gyp` install-time execution path, but changed public
markers from the older Miasma/Hades strings to `Alright Lets See If This Works` and
`RevokeAndItGoesKaboom`. JFrog also published decoded strings for a gated `SEED_PAT` bootstrap path,
two RSA public-key prefixes, Bun v1.3.13/v1.3.14 download paths, GitHub dead-drop/result artifacts,
and AI/editor persistence targets such as `.windsurfrules`, `.github/copilot-instructions.md`,
`mcp.json`, and `.aider.conf.yml`.

## Notes

- The Red Hat npm reports did not publish file hashes; the Hades PyPI report published SHA-256 hashes
  for `_index.js` variants and the shared `*-setup.pth` loader, so `knownSha256` now carries those
  PyPI hashes.
- `suspiciousFilenames` is intentionally empty: the install artifact is `index.js`/`_index.js`, far
  too generic to flag by name. The named persistence files (`kitty-monitor*`, `gh-token-monitor*`) are
  already carried by `mini-shai-hulud-2026-05`; they appear here only as composite signals to avoid
  duplicate findings.
- The JS-payload and `.github/workflows/*.yml` IOCs/composites only fire where those files are
  content-scanned. The app content-scans GitHub workflow YAML in the default profile and now also
  content-scans repo-open trigger paths (`.claude/settings.json`, `.gemini/settings.json`,
  `.cursor/rules/*`, `.vscode/tasks.json`, `.github/setup.js`/`.mjs`) so the Microsoft/Azure
  repo-injection variant is visible in normal project scans. Other broad JavaScript payload hunting
  remains a deep or Shai-Hulud profile concern.
- The app needs `.pth` files and `_index.js` package-cache files to be eligible for text/hash scanning
  so the Hades loader strings and published member hashes fire inside wheels and installed
  site-packages.
